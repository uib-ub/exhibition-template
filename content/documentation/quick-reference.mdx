# Quick Reference Guide

This guide provides quick access to common tasks and code snippets you'll use while creating your exhibition.

## Basic Page Structure

Every page should start with frontmatter and a main heading:

```markdown
---
title: Your Page Title
description: A brief description of the page content
keywords: 
  - keyword1
  - keyword2
  - keyword3
---

# Main Page Heading

Your content goes here...
```

### Frontmatter Options

The frontmatter section (between `---` markers) supports several fields:

- **`title`** (required): The page title that appears in the browser tab and navigation
- **`description`** (optional): A brief description of the page content, used for SEO
- **`keywords`** (optional): A list of keywords related to the page content, used for SEO

### Frontmatter Examples

**Simple page:**
```markdown
---
title: About Us
---
```

**Page with SEO metadata:**
```markdown
---
title: Medieval Artifacts
description: Explore our collection of medieval artifacts from the 12th century
keywords:
  - medieval
  - artifacts
  - 12th century
  - history
---
```

## Work Component

### Basic Usage
```tsx

<Work manifest="object-id">
  <Work.Content>
```

## Common Tasks

### Adding a New Page
1. Create a new `.mdx` file in the appropriate folder
2. Add frontmatter with a title
3. Write your content
4. Save and commit your changes

### Adding Images
1. Upload image to `public/` folder
2. Reference it in your content: `![Description](/image-name.jpg)`
3. Use descriptive file names
4. Keep file sizes under 2MB

### Creating Links Between Pages
- Internal pages: `[Link text](/page-name)`
- External sites: `[Link text](https://example.com)`
- Same page sections: `[Link text](#section-name)`

### Making Text Stand Out
- **Bold** for emphasis: `**text**`
- *Italic* for subtle emphasis: `*text*`
- `Code` for technical terms: `` `text` ``
- Headings for structure: `# Heading`

## Keyboard Shortcuts

### In Codespace
- `Ctrl+Z` (or `Cmd+Z`) - Undo
- `Ctrl+F` (or `Cmd+F`) - Find
- `Ctrl+H` (or `Cmd+H`) - Replace
- `Ctrl+S` (or `Cmd+S`) - Save file (usually not needed due to autosave)

### In Browser
- `F5` (or `Ctrl+R`) - Refresh page
- `Ctrl+F5` (or `Cmd+Shift+R`) - Hard refresh
- `Ctrl+Shift+I` - Open developer tools

## Best Practices

### Content
- Write in plain language
- Use short paragraphs
- Include descriptive headings
- Test your links

### Images
- Use descriptive file names
- Compress large images
- Include alt text
- Use appropriate formats

### Code
- Copy examples exactly
- Check spelling carefully
- Test one change at a time
- Save frequently

### Organization
- Use consistent naming
- Group related content
- Create logical navigation
- Keep structure simple

Remember: When in doubt, start simple and build complexity gradually!
