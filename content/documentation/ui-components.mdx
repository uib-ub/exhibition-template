# UI Components Guide

This guide shows you how to use the built-in design elements to make your exhibition look professional and polished. These components are like pre-designed building blocks that you can arrange to create your perfect exhibition.

## What are UI Components?

UI components are pre-built design elements that you can add to your own custom components. They are built with [Shadcn UI](https://ui.shadcn.com/) and [Tailwind CSS](https://tailwindcss.com/). 

## Buttons - Interactive Elements

Buttons are clickable elements that can link to other pages or perform actions. They come in different styles to match your exhibition's theme.

### Basic Button

```tsx
import { Button } from '@/components/ui/button'

<Button>Click me</Button>
```

### Button Styles (Variants)

Choose from different button styles to match your content:

<div className="flex gap-2 flex-wrap my-3">
  <Button variant="default">Default</Button>
  <Button variant="destructive">Destructive</Button>
  <Button variant="outline">Outline</Button>
  <Button variant="secondary">Secondary</Button>
  <Button variant="ghost">Ghost</Button>
  <Button variant="link">Link</Button>
</div>

**When to use each style:**
- **Default** - Primary actions (like "Learn More" or "View Exhibition")
- **Destructive** - Dangerous actions (like "Delete" or "Remove")
- **Outline** - Secondary actions that are still important
- **Secondary** - Less important actions
- **Ghost** - Subtle actions that don't draw too much attention
- **Link** - Actions that look like regular text links

### Button Sizes

Buttons come in different sizes to fit different contexts:

<div className="flex gap-2 flex-wrap my-3 items-center">
  <Button size="sm">Small</Button>
  <Button size="default">Default</Button>
  <Button size="lg">Large</Button>
  <Button size="icon"><IconUsers /></Button>
</div>

**When to use each size:**
- **Small** - When space is limited or for minor actions
- **Default** - Most common size for regular actions
- **Large** - For important calls-to-action or when you want to draw attention
- **Icon** - For buttons that only contain an icon (like a menu button)

### Practical Examples

**Navigation button:**
```tsx
<Button variant="outline" size="lg">
  <IconArrowRight />
  Continue to Next Section
</Button>
```

**Action button:**
```tsx
<Button variant="default">
  View Full Image
</Button>
```

**Subtle link:**
```tsx
<Button variant="link">
  Read More About This Artifact
</Button>
```

## How to Use These Components

### Step 1: Create a new component

Create a new file in the `src/components/` folder. For example, if you want to create a new Callout component, create a new file called `callout.tsx` in the `src/components/` folder.

### Step 2: Import the UI Component

At the top of your file, add the import statement:

```tsx
import { Button } from '@/components/ui/button'
import { IconHome } from '@tabler/icons-react'
```

### Step 3: Add the new Component to Your Content

Place the component where you want it to appear in your content:

```tsx
import { Callout } from '@/components/callout'

<Callout>
  <Callout.Title>
    Welcome to My Exhibition
  </Callout.Title>
  <Callout.Content>
    This is my amazing exhibition about medieval manuscripts.
  </Callout.Content>
</Callout>
```

## Design Tips for Non-Designers

### Color and Style
- **Be consistent** - Use the same button styles throughout your exhibition
- **Use contrast** - Make sure buttons stand out from the background
- **Don't overdo it** - Too many different styles can look cluttered

### Layout
- **Group related elements** - Put navigation buttons together
- **Use whitespace** - Give elements room to breathe
- **Align elements** - Keep things neat and organized

### Accessibility
- **Use descriptive text** - Don't rely only on icons
- **Provide alternatives** - Some users can't see icons
- **Test your design** - Make sure it works on different devices
